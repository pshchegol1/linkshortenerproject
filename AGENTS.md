# Agent Instructions Reference



ALL LLM agent coding standards and instructions for this project are maintained in the `/docs` directory.

**It is INCREDIBLY IMPORTANT to ALWAYS read the relevant individual instructions files within the `/docs` directory BEFORE generating ANY code.**

- See `/docs/agent-instructions.md` for general guidelines and standards.
- Additional topic-specific instructions should be added as separate markdown files in `/docs` (e.g., `database.md`, `ui.md`, `api.md`).

- **UI Standards:** See `/docs/ui.md` for rules on using shadcn/ui for all UI elements. Do not create custom UI components; always use shadcn/ui components.

- **Authentication:** See `/docs/auth.md` for all authentication requirements and standards (Clerk-only, protected routes, modal flows, etc).


Agents must adhere to these standards when contributing code or documentation to this project.

**ALWAYS refer to and carefully read the relevant .md instruction files in `/docs` BEFORE generating ANY code. Failure to do so may result in non-compliant or incorrect contributions.**
